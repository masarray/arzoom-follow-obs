Troubleshooting

Fix the most common ArZoom setup problems.

Start with the symptom you see. Most problems are caused by an incomplete install, an older mixed-version package, an unassigned hotkey, or incorrect monitor mapping.

The ArZoom filter does not appear

  1. Confirm you are adding an Effect Filter to a video source.
  2. Close OBS completely.
  3. Reinstall the complete ArZoom package.
  4. Restart OBS normally, not in Safe Mode.
  5. Check Help → Log Files → View Current Log and search for ArZoom.

The plugin DLL and data directory must both be present. A DLL without the shader resources is not a complete installation.

The filter exists, but says “No properties available”

This behavior was fixed in v0.1.2. Remove the empty filter, install the newest release, restart OBS, and add the filter again.

The hotkey row does not appear

  1. Confirm the ArZoom filter controls are visible.
  2. Restart OBS after installation.
  3. Open Settings → Hotkeys.
  4. Clear both Hotkeys search fields.
  5. Search for ArZoom in the text filter.

The expected row is:

ArZoom — Toggle Zoom & Mouse Follow

v0.1.3 and newer register this as one global frontend hotkey.

The hotkey disappears after restarting OBS

Install v0.1.4 or newer. Then:

  1. Assign the ArZoom shortcut.
  2. Click Apply or OK in OBS Settings.
  3. Return to the ArZoom filter.
  4. Click Save current hotkey now.
  5. Close OBS normally and reopen it.

ArZoom v0.1.4 explicitly stores and reloads the binding from the active OBS profile.

The hotkey is visible but does nothing

  • Check that the ArZoom filter is enabled using the eye icon.
  • Confirm the Display Capture source is visible in the active scene.
  • Try the Test zoom / toggle button inside the filter.
  • Use a different shortcut to rule out a conflict.
  • Check whether OBS hotkeys are disabled while another application has focus by your OBS Hotkey Focus Behavior setting.

Zoom works, but the viewport does not follow the mouse

  1. Set Mouse follow to Smart or Centered.
  2. Set Target monitor to Auto.
  3. Confirm the filter is attached to the intended Display Capture source.
  4. If Auto mapping is wrong, manually select the captured monitor.

Fixed mode intentionally does not follow new cursor positions.

The frame stops moving near an edge

This is expected and protects the video output. Near a side or corner, it is impossible to keep the cursor centered while also keeping the viewport inside the captured monitor. ArZoom prioritizes valid pixels and stops at the nearest legal viewport position.

The motion feels too active

  • Use Smart follow.
  • Increase the Stable safe zone.
  • Use Smooth movement.
  • Reduce Zoom amount.

For long tutorials, start around 1.70×–2.00× with a 28%–35% safe zone.

The motion feels too slow

  • Change Movement from Smooth to Balanced or Fast.
  • Reduce the Stable safe zone if the viewport waits too long before following.
  • Avoid an excessive zoom level, because a smaller visible viewport requires more panning.

OBS shows a black frame or the source disappears

ArZoom is designed to fail through to the original source, so a black frame is not expected. Immediately disable the filter using the eye icon, then:

  1. Confirm the full shader/data folder was installed.
  2. Check the OBS log for ArZoom, effect, or shader errors.
  3. Remove older ArZoom DLL/data copies from other OBS plugin paths.
  4. Reinstall one matching version of the complete package.
For a live event: switch to a fallback scene without ArZoom before investigating further.

The plugin is installed twice

Portable OBS and system-installed OBS can load plugins from different directories. Search for duplicate arzoom.dll and data/obs-plugins/arzoom locations. Keep only the matching package used by the OBS instance you launch.

Mixed-DPI or multi-monitor mapping is wrong

  • Use the monitor selector instead of Auto.
  • Restart OBS after changing Windows display arrangement or scale.
  • Test the cursor in every corner of the selected monitor.
  • Include your monitor layout, resolutions, scale percentages, and OBS version in a bug report.

What to include in a bug report

  • ArZoom version
  • OBS Studio version
  • Windows version
  • GPU model
  • Display resolutions and Windows scale percentages
  • Display Capture method and selected monitor
  • Exact steps to reproduce
  • Relevant OBS log section
  • A short recording or screenshot when safe to share

Do not publish confidential screen content, stream keys, passwords, private URLs, or customer information.
