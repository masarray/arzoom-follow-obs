Privacy and security

Local processing. No telemetry. No cloud dependency.

ArZoom performs its zoom, cursor mapping, animation, and video effect locally inside OBS Studio.

Data ArZoom uses locally

  • The current mouse position while zoom-follow is active
  • The selected Display Capture dimensions and monitor coordinates
  • ArZoom filter settings stored by OBS
  • The configured ArZoom hotkey stored in the active OBS profile

This data is used only to calculate the on-screen zoom and viewport position.

Data ArZoom does not transmit

  • Captured video frames
  • Cursor coordinates
  • Hotkeys
  • OBS scenes, profiles, or source names
  • Application usage analytics
  • Hardware identifiers
  • Personal information

No external service requirement

ArZoom does not require a user account, license server, analytics platform, advertising network, cloud API, OCR service, or AI inference provider. The runtime plugin is designed to function without an internet connection.

Open-source inspection

The source code is publicly available on GitHub. Users and contributors can inspect how the plugin reads cursor position, stores settings, renders the video filter, and handles failure conditions.

Security reports

Do not post a suspected vulnerability publicly before the maintainer has had a reasonable opportunity to investigate. Follow the instructions in the repository Security Policy.

Important OBS reminder

ArZoom changes how a Display Capture is framed; it does not redact confidential information. Anything visible inside the captured monitor may still be recorded or streamed. Protect passwords, notifications, private documents, customer data, and stream keys using appropriate OBS scenes and Windows privacy controls.
