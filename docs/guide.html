Beginner setup guide

Install ArZoom and create your first smooth screen zoom.

This guide assumes you already have OBS Studio and a Display Capture source. No Python, scripts, or programming knowledge is required.

Preview notice: ArZoom v0.1.4 is a Windows preview. Test it before an important production livestream and keep a normal OBS scene available as a backup.

Before you begin

  • Windows 10 or Windows 11, 64-bit
  • OBS Studio with a working Display Capture source
  • Permission to install files into your OBS installation
  • The latest ArZoom Windows installer from GitHub Releases

1. Download ArZoom

Open ArZoom Releases. Download the newest Windows installer. Prefer the installer over the manual ZIP unless you specifically use a portable OBS installation.

2. Close OBS completely

Exit OBS before installing. Check the Windows system tray as well—OBS must not still be running in the background, because Windows cannot reliably replace a plugin DLL that is currently loaded.

3. Run the installer

Start the ArZoom installer and select your OBS installation directory if it is not detected automatically. The package must install both:

obs-plugins/64bit/arzoom.dll
data/obs-plugins/arzoom/...

The DLL and data folder are both required. The data folder contains the shader and interface translations.

4. Add ArZoom to Display Capture

  1. Start OBS.
  2. In Sources, select the Display Capture that shows the monitor you want to zoom.
  3. Right-click the source and choose Filters.
  4. Under Effect Filters, click +.
  5. Select ArZoom - Smart Mouse Zoom.

The right side of the Filters window should now show ArZoom controls. If it says “No properties available,” remove the filter and install v0.1.2 or newer.

5. Assign the zoom hotkey

  1. Inside the ArZoom filter panel, click Open OBS Hotkeys Settings.
  2. Find ArZoom — Toggle Zoom & Mouse Follow.
  3. Click the empty input field.
  4. Press your preferred shortcut, for example Ctrl + ~.
  5. Click Apply or OK.
Choose a unique shortcut. Avoid a key combination already used by recording, streaming, scene switching, or the application you are demonstrating.

ArZoom v0.1.4 explicitly saves and restores this binding when OBS restarts and when the active OBS profile changes. The filter also includes a Save current hotkey now button for manual confirmation.

6. Use the recommended first settings

SettingStart withWhy
Zoom amount2.00×Large enough for menus and text without feeling extreme.
Mouse followSmartKeeps the view stable during small cursor movements.
MovementSmoothComfortable for tutorials and longer explanations.
Stable safe zone28%Reduces constant micro-panning.
Target monitorAutoUses the monitor selected in Display Capture.

7. Test the complete motion

  1. Move the mouse near the center of the captured monitor.
  2. Press your ArZoom hotkey once.
  3. Wait for the zoom-in to settle.
  4. Move the cursor slowly around the safe zone.
  5. Move it toward all four screen edges.
  6. Press the same hotkey again.

The view should zoom in smoothly, remain stable during small mouse movement, follow when the cursor leaves the safe zone, stop at valid monitor edges, then recenter and return to the full screen.

How the follow modes differ

Smart — recommended

The viewport remains still while the cursor stays inside a stable central zone. This is usually the most comfortable mode for viewers.

Centered

The viewport tries to keep the cursor closer to the configured anchor. It reacts more often and may feel more active.

Fixed

ArZoom changes magnification but does not follow new cursor positions. Use it when you want a static close-up.

Choosing the zoom amount

  • 1.40×–1.70×: subtle presentation zoom with minimal motion.
  • 1.80×–2.20×: recommended for most software tutorials.
  • 2.30×–3.00×: small controls, dense engineering software, or high-resolution monitors.
  • Above 3.00×: use carefully; the visible area becomes small and viewport movement becomes more noticeable.

Before recording or going live

  • Perform a short local recording and watch it at normal speed.
  • Check that your shortcut does not conflict with the demonstrated application.
  • Test every monitor used by Display Capture.
  • Confirm the hotkey after restarting OBS.
  • Check OBS Stats for rendering or encoding lag.
  • Keep a fallback scene without ArZoom for important live events.

Next steps

When the basic setup works, adjust only one parameter at a time. Start with Zoom amount, then Stable safe zone, then Movement. Most users do not need to change the advanced anchor controls.
