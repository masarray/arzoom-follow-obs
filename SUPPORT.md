# ArZoom Support

ArZoom is an open-source community project. Support is provided on a best-effort basis through GitHub.

## Start here

1. Read the [beginner setup guide](https://masarray.github.io/arzoom-follow-obs/guide.html).
2. Check the [troubleshooting guide](https://masarray.github.io/arzoom-follow-obs/troubleshooting.html).
3. Install the newest release and restart OBS completely.
4. Search existing [GitHub issues](https://github.com/masarray/arzoom-follow-obs/issues) before opening a new one.

## Current supported scope

The current preview focuses on:

- Windows 10/11 x64
- OBS Studio Display Capture
- one global toggle hotkey
- Smart, Centered, and Fixed zoom-follow modes
- single- and multi-monitor desktop layouts

Window Capture, Game Capture, macOS, and Linux are not part of the current MVP.

## Reporting a bug

Use the bug report form and include:

- ArZoom version
- OBS Studio version
- Windows version
- GPU model
- monitor resolutions and Windows scaling percentages
- Display Capture method and selected monitor
- exact reproduction steps
- relevant OBS log lines
- screenshots or a short recording when safe to share

Never post stream keys, passwords, confidential customer screens, private URLs, or proprietary data.

## Feature requests

Feature requests are welcome when they support ArZoom's core goal: smooth, reliable, beginner-friendly screen zoom for recording and livestreaming. Requests that significantly increase runtime complexity or motion instability may be deferred.

## Response expectations

There is no guaranteed response time or service-level agreement. Clear, reproducible reports with logs and environment details are the easiest to investigate.
